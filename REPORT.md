# Эмпирическое исследование алгоритмов сортировки строк

## Этап 1. Подготовка тестовых данных

Реализация: StringGenerator (string_generator.h/.cpp). Отсавил число енерации std::mt19937 для точности анализа

## Этап 2. Стандартные сортировки

Реализованы quickSort и mergeSort. Cравнения сиволов считаются в compare_strings (общий счётчик g_char_comparisons).

## Замеры

Запуск:

```
make run
./stringsort_bench --repeats 5 --out r.csv
```

Формат results.csv:

```
algorithm,array_type,n,avg_time_us,avg_char_comparisons
```

## Результаты

Сырые данные: data/results.csv.  
Графики: data/plots/ — по одному графику "время и кол-во" и "сравнения и кол-во" на каждый из трёх типов массивов.

## Выводы

Ожидаемые соотношения:

- На случайных данных StringMergeSort экономит сравнения относительно mergeSort за счёт LCP (наблюдалось ~33%).
- тернарный quick sort и MSD radix лучше работают на длинных общих префиксах.
- MSB radix с переключением быстрее по времени, чем MSDRadixSort без переключения, на малых данных.
- На обратной строке стандартный quick sort не работает с асимптотикой O(n^2), но даёт больше сравнений, чем merge sort.
